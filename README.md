# BlockchainNews

区块链资讯报纸仓库。当前产出模式为周报 + 月度总结：周报负责技术、产业、安全、社会、市场与研究的精选详解；月度总结只保留科技新闻总结与安全事故总结，并提高事实密度。

## 目录规则

```text
BlockchainNews/
  2026年3月刊/
    Week1/
      2026-03-01至2026-03-07周报.md
    Week2/
      2026-03-08至2026-03-14周报.md
    ...
    月度总结.md
```

- `Week1`: 每月 `1-7` 日
- `Week2`: 每月 `8-14` 日
- `Week3`: 每月 `15-21` 日
- `Week4`: 每月 `22-28` 日
- `Week5`: 每月 `29-月末`
- 跨月周报按 `period_end` 所在月份归档。例如 `2026-03-29至2026-04-04周报.md` 放入 `2026年4月刊/Week1/`。

## 周报标准

每周日北京时间 `07:00` 生成上一完整周，即上周日 `00:00` 至周六 `23:59`。标题固定为：

```markdown
# 区块链周报（YYYY-MM-DD 至 YYYY-MM-DD）
```

周报必须包含以下板块：

1. `导读`
2. `区块链技术与产业`
3. `区块链安全`
4. `区块链与社会`
5. `加密市场与宏观`
6. `技术研究`
7. `后续关注`

写作要求：

- 从概述改为详解，精选而不是堆砌；五个正文栏目各保留 `3-5` 条。
- 每条新闻要优先贴近原始来源：来源事实清楚时，先翻译、保留并适度优化原文关键信息，再补充必要的前因后果、关键术语、技术机制和影响。不要用统一三段式套模板，不要写来源标签句，也不要只做标题扩写。
- `区块链安全` 只放漏洞、攻击、审计、防护、复盘与安全工程进展；法律、监管、政治、社会争议必须放到 `区块链与社会`。
- 所有可报道条目统一使用 `###` 作为条目标题。
- 非研究条目必须使用结构化来源行：`**来源：** [来源名](URL) | YYYY-MM-DD`，来源日期必须位于周报时间窗内。
- 非研究条目通常写 `2-5` 个自然段，按内容复杂度展开；禁止单句简讯和机械三段式。
- 不采用原创图、AI 生成图或自制示意图。若原始来源提供了可引用的图表、协议图、攻击路径图或报告图，可放入同目录 `imgs/` 下，并紧跟来源说明；没有合适来源图时直接不插图。
- 所有条目都必须与区块链产业相关。

## 技术研究

- `技术研究` 可收录论文、官方研究报告和高质量第三方技术分析。
- 每期 `3-5` 条，每条都必须深读，不再使用一两句摘要凑数。
- `原文链接` 必须指向原始出处，不能指向转载页、媒体页、社区页、聚合页或搜索页。
- `摘要速写` 不能用 `这篇论文`、`论文提出`、`该研究` 等模板化开头。
- 每条必须包含 `原文链接`、`原始发表`、`摘要速写`、`核心贡献`，随后用连续段落做中性的学术内容介绍。不要固定写成 `背景与问题`、`方法/机制`、`关键发现`、`局限与后续` 四个小标题。
- 技术研究写作时站在专业学者视角，只介绍论文/报告/技术文章本身：研究对象、必要术语、方法细节、数据或协议假设、实验或案例、结果以及作者给出的边界。不要加入写作者自己的观点、建议或价值判断。
- 禁止使用伪审稿套话、读者引导、价值判断或条件式评价。正文只介绍论文/报告/技术文章自身内容，不评价其用途，不替读者下判断。

固定结构：

```markdown
### 《Title》

- 原文链接：https://example.com/original
- 原始发表：YYYY-MM-DD
- 摘要速写：从具体系统、机制、攻击、结果或影响切入的一句话摘要。
- 核心贡献：
  - 贡献 1
  - 贡献 2
  - 贡献 3

随后按论文、报告或文章自身逻辑写自然段，介绍研究对象、必要术语、方法/数据/协议机制、实验或案例、结果和作者给出的边界。不要写作者之外的观点、建议或价值判断。
```

## 月度总结

次月 1 日北京时间 `07:30` 生成上一自然月总结。标题固定为：

```markdown
# 区块链月度总结（YYYY-MM）
```

月度总结只允许两个顶层模块：

1. `科技新闻总结`
2. `安全事故总结`

要求：

- 科技新闻总结写月度技术主线、关键项目、机制变化和产业影响，不写泛行情。
- 安全事故总结按高精度安全月报标准写日期、项目、链、损失金额或 `未披露`、攻击向量、根因、资金状态、修复动作。
- 月报可以复盘周报已报道事件，但不得复制周报行文。
- 月报不使用原创图；仅在有清晰来源图表时引用并标注来源。

## 去重与校验

- 周报候选需与前 `8` 期周报、当前月已覆盖条目、最近 `2` 期月报做去重。
- 研究原文链接 `90` 天内不得重复。
- 同一原文链接是硬重复；高相似标题、同一项目同一事件窗口或重复摘要骨架必须替换。
- 生成后必须运行：

```bash
python3 ~/.openclaw/workspace/skills/blockchain-newsroom/scripts/verify_issue_accuracy.py --issue-file /absolute/path/to/publication.md
```

- 校验失败不得 commit、push 或推送公众号草稿。
