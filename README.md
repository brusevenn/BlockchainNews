# BlockchainNews

区块链资讯报纸仓库。目标是用简洁但专业的中文 Markdown 产出可日更、可周更、可月更、可年更的技术型资讯刊物。

## 目录规则

```text
BlockchainNews/
  2026年3月刊/
    Week1/
      2026-03-01.md
      ...
      2026-03-07.md
      本周总结.md
    Week2/
    ...
    月度总结.md
  2026年年度总结.md
```

- `Week1`: 每月 `1-7` 日
- `Week2`: 每月 `8-14` 日
- `Week3`: 每月 `15-21` 日
- `Week4`: 每月 `22-28` 日
- `Week5`: 每月 `29-月末`

凌晨 `02:00` 生成的是**前一天**的日报。例如 `2026-03-08 02:00` 生成 `2026-03-07.md`，路径应为 `2026年3月刊/Week1/2026-03-07.md`。

## 日报标准

日报标题固定为：

```markdown
# 每日区块链资讯速读（YYYY-MM-DD）
```

日报必须包含以下板块：

1. `导读`
2. `区块链技术与产业`
3. `区块链安全`
4. `加密市场与宏观`
5. `新论文与研究报告`
6. `后续关注`

写作要求：

- 以技术新闻、安全事件、研究进展为主，不写空泛行情口水稿。
- 每条新闻要讲清楚前因后果、技术点和影响。
- 非论文条目在正文中写 `据 xxx 报道/披露` 即可，不强制附原始链接。
- 论文和研究报告必须附原文链接，并总结摘要与核心贡献。
- 控制到一个正常上班前十分钟内能读完，但重大新闻可以适度展开。

## 总结类文件

- `WeekN/本周总结.md`：连接同周事件，强调时间线和主线演化。
- `YYYY年M月刊/月度总结.md`：回顾技术、安全、市场、研究四条主线。
- `YYYY年年度总结.md`：年度复盘，可加入榜单和年度时间线。
