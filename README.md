# BlockchainNews

区块链资讯报纸仓库。当前产出模式为周报 + 月度总结 + 优质转载。OpenClaw 的角色是优质内容的转载者/编译者，不是原创作者；周报以来源原文的中文翻译、搬运和整理为主，尽量减少原创评论。月度总结只保留科技新闻总结与安全事故总结，并提高事实密度。

## 目录规则

```text
BlockchainNews/
  2026年3月刊/
    Week1/
      2026-03-01至2026-03-07周报.md
    Week2/
      2026-03-08至2026-03-14周报.md
    ...
    月度总结.md
  优质转载/
    2026年4月/
      文章标题.md
```

- `Week1`: 每月 `1-7` 日
- `Week2`: 每月 `8-14` 日
- `Week3`: 每月 `15-21` 日
- `Week4`: 每月 `22-28` 日
- `Week5`: 每月 `29-月末`
- 跨月周报按 `period_end` 所在月份归档。例如 `2026-03-29至2026-04-04周报.md` 放入 `2026年4月刊/Week1/`。

## 周报标准

每周日北京时间 `07:00` 生成上一完整周，即上周日 `00:00` 至周六 `23:59`。标题固定为：

```markdown
# 区块链周报（YYYY-MM-DD 至 YYYY-MM-DD）
```

周报必须包含以下板块：

1. `导读`
2. `区块产业发展`
3. `区块链安全事件`
4. `区块链与社会`

写作要求：

- 从概述改为转载/编译，精选而不是堆砌；三个正文栏目各保留约 `10` 条，实际控制在 `8-12` 条。
- 每条新闻要优先贴近原始来源：来源事实清楚时，先翻译、保留并适度优化原文关键信息，再补充必要的前因后果、关键术语、技术机制和影响。不要用统一三段式套模板，不要写来源标签句，也不要只做标题扩写。
- `区块产业发展` 聚焦区块链技术本身密切相关的技术进展、基础设施、新应用和产业采用。
- `区块链安全事件` 聚焦攻击案例分析、安全漏洞、防护措施、审计和安全复盘。
- `区块链与社会` 聚焦各国监管政策、执法、制裁、机构采用和虚拟货币市场动态。
- 所有可报道条目统一使用 `###` 作为条目标题。
- 周报条目必须使用结构化来源行：`**来源：** [来源名](URL) | YYYY-MM-DD`，来源日期必须位于周报时间窗内。
- 条目通常写 `2-5` 个自然段，按来源内容复杂度展开；禁止单句简讯、机械三段式和原创观点句。
- 不采用原创图、AI 生成图或自制示意图。若原始来源提供了可引用的图表、协议图、攻击路径图或报告图，可放入同目录 `imgs/` 下，并紧跟来源说明；没有合适来源图时直接不插图。
- 所有条目都必须与区块链产业相关。

旧栏目 `技术研究`、`加密市场与宏观`、`后续关注` 不再用于周报。

## 优质转载

OpenClaw 需要不定期发掘并积累机构官网、项目博客、安全公司、治理论坛、X、个人技术博客等来源，优先选择适合私人学习的高质量文章。候选来源要写入长期记忆和候选池，记录 URL、来源、作者/机构、日期、主题、适合栏目和推荐理由。

优质转载固定结构：

```markdown
# 中文标题

- 原文链接：https://example.com/original
- 来源：Source Name
- 作者/机构：Author or Organization
- 原始发布时间：YYYY-MM-DD
- 仅供个人学习：本文为私人学习用途的中文学习稿。

## 中文学习稿

按来源文章逻辑进行中文翻译/编译，保留来源事实、结构和关键术语解释，不加入原创观点。
```

## 月度总结

次月 1 日北京时间 `07:30` 生成上一自然月总结。标题固定为：

```markdown
# 区块链月度总结（YYYY-MM）
```

月度总结只允许两个顶层模块：

1. `科技新闻总结`
2. `安全事故总结`

要求：

- 科技新闻总结写月度技术主线、关键项目、机制变化和产业影响，不写泛行情。
- 安全事故总结按高精度安全月报标准写日期、项目、链、损失金额或 `未披露`、攻击向量、根因、资金状态、修复动作。
- 月报可以复盘周报已报道事件，但不得复制周报行文。
- 月报不使用原创图；仅在有清晰来源图表时引用并标注来源。

## 去重与校验

- 周报候选需与前 `8` 期周报、当前月已覆盖条目、最近 `2` 期月报做去重。
- 优质转载原文链接 `90` 天内不得重复。
- 同一原文链接是硬重复；高相似标题、同一项目同一事件窗口或重复摘要骨架必须替换。
- 生成后必须运行：

```bash
python3 ~/.openclaw/workspace/skills/blockchain-newsroom/scripts/verify_issue_accuracy.py --issue-file /absolute/path/to/publication.md
```

- 校验失败不得 commit、push 或推送公众号草稿。
