# BlockchainNews

区块链资讯报纸仓库。目标是用简洁但专业的中文 Markdown 产出可日更、可周更、可月更、可年更的技术型资讯刊物。

## 目录规则

```text
BlockchainNews/
  2026年3月刊/
    Week1/
      2026-03-01.md
      ...
      2026-03-07.md
      本周总结.md
    Week2/
    ...
    月度总结.md
  2026年年度总结.md
```

- `Week1`: 每月 `1-7` 日
- `Week2`: 每月 `8-14` 日
- `Week3`: 每月 `15-21` 日
- `Week4`: 每月 `22-28` 日
- `Week5`: 每月 `29-月末`

凌晨 `02:00` 生成的是**前一天**的日报。例如 `2026-03-08 02:00` 生成 `2026-03-07.md`，路径应为 `2026年3月刊/Week1/2026-03-07.md`。

## 日报标准

日报标题固定为：

```markdown
# 每日区块链资讯速读（YYYY-MM-DD）
```

日报必须包含以下板块：

1. `导读`
2. `区块链技术与产业`
3. `区块链安全`
4. `区块链与社会`
5. `加密市场与宏观`
6. `新论文与研究报告`
7. `后续关注`

写作要求：

- 以技术新闻、安全事件、研究进展为主，不写空泛行情口水稿。
- 每条新闻要讲清楚前因后果、技术点和影响。
- `区块链安全` 只放漏洞、攻击、审计、防护、复盘与安全工程进展。法律、监管、政治、社会争议必须单列到 `区块链与社会`。
- 非论文条目在正文中写 `据 xxx 报道/披露` 即可，不强制附原始链接。
- 论文和研究报告必须附原文链接，并总结摘要与核心贡献。
- 控制到一个正常上班前十分钟内能读完，但重大新闻可以适度展开。
- 日报末尾必须附上 `重复项检查结果`，说明本次检测到的疑似重复条数与处理结果。

## 去重与历史比对

- 每日候选新闻在成稿前，必须与过去 `7` 天日报以及上月 `月度总结.md` 做历史比对。
- 如果两条内容在事件主体、发生时间、关键事实和影响上高度一致，或语义相似度超过 `90%`，应标记为 `疑似重复`。
- 对于同一事件的跨源报道，只保留一个条目，优先级依次为：
  1. 官方公告、事故复盘、治理提案、论文原文
  2. 安全/取证/研究机构
  3. 主流科技或加密媒体
  4. 中文社区媒体
  5. X 帖子、newsletter、聚合器
- 若某条新闻属于同一事件的新进展，必须在正文里明确写出“这是对前次事件的新增进展”，不能当作全新新闻重复刊发。

## 学术来源范围

- `新论文与研究报告` 不得长期被行业报告占满；在存在合适论文或预印本时，优先收录论文。
- 论文检索必须覆盖官方 CCF 国际学术会议/期刊目录中的相关 A/B 类 venue，至少包含这些类别页：
  - `https://www.ccf.org.cn/Academic_Evaluation/ARCH_DCP_SS/`
  - `https://www.ccf.org.cn/Academic_Evaluation/CN/`
  - `https://www.ccf.org.cn/Academic_Evaluation/NIS/`
  - `https://www.ccf.org.cn/Academic_Evaluation/Cross_Compre_Emerging/`
- 预印本检索必须覆盖：
  - `https://arxiv.org/`，重点关注 `cs.CR`、`cs.NI`、`cs.DC`
  - `https://eprint.iacr.org/`
- 可补充 `OpenReview` 与研究导向的 `SSRN`，但不得用泛行业评论替代论文。
- 每条论文/报告条目都必须包含原文链接、摘要速写和 `3` 条核心贡献。

## 总结类文件

- `WeekN/本周总结.md`：必须比日刊更详细，连接同周事件，强调时间线、主线演化和连续性。
- `YYYY年M月刊/月度总结.md`：必须加入月内时间线，系统回顾技术、安全、市场、研究四条主线。
- `YYYY年年度总结.md`：年度复盘应是最详细的一层，可加入榜单、时间线和主题盘点。
