# BlockchainNews

区块链资讯报纸仓库。目标是用简洁但专业的中文 Markdown 产出可日更、可周更、可月更、可年更的技术型资讯刊物。

## 目录规则

```text
BlockchainNews/
  2026年3月刊/
    Week1/
      2026-03-01.md
      ...
      2026-03-07.md
      本周总结.md
    Week2/
    ...
    月度总结.md
  2026年年度总结.md
```

- `Week1`: 每月 `1-7` 日
- `Week2`: 每月 `8-14` 日
- `Week3`: 每月 `15-21` 日
- `Week4`: 每月 `22-28` 日
- `Week5`: 每月 `29-月末`

凌晨 `02:00` 生成的是**前一天**的日报。例如 `2026-03-08 02:00` 生成 `2026-03-07.md`，路径应为 `2026年3月刊/Week1/2026-03-07.md`。

## 日报标准

日报标题固定为：

```markdown
# 区块链日报（YYYY-MM-DD）
```

日报必须包含以下板块：

1. `导读`
2. `区块链技术与产业`
3. `区块链安全`
4. `区块链与社会`
5. `加密市场与宏观`
6. `技术研究`
7. `后续关注`

写作要求：

- 以技术新闻、安全事件、研究进展为主，不写空泛行情口水稿。
- 每条新闻要讲清楚前因后果、技术点和影响。
- `区块链安全` 只放漏洞、攻击、审计、防护、复盘与安全工程进展。法律、监管、政治、社会争议必须单列到 `区块链与社会`。
- 所有可报道条目统一使用 `###` 作为条目标题。
- `区块链技术与产业`、`区块链安全`、`技术研究` 每个栏目至少 `5` 条。
- 非研究条目必须使用结构化来源行：`**来源：** [来源名](URL) | YYYY-MM-DD`。
- `区块链技术与产业` 与 `区块链安全` 的来源日期必须落在 `T-3` 到 `T`；`T` 取日报文件日期。
- `技术研究` 接受论文/预印本、官方研究报告、高质量第三方技术分析，但正文中的 `原文链接` 必须指向原始出处，不能指向转载页或社区页。
- `技术研究` 条目必须附 `原文链接`、`原始发表`、`摘要速写` 和 `核心贡献`。
- `核心贡献` 必须单独起一行，并使用二级列表分 `3` 点描述；这个格式同时适用于 GitHub 日报源 Markdown 和公众号日报成品。
- 公众号成稿里，`原文链接`、`原始发表`、`摘要速写`、`核心贡献` 四个标签必须保持同一级视觉层级；其中 `核心贡献` 的正文部分再以下级列表分 `3` 点展开。
- 所有可报道栏目都必须先和过去 `7` 天日报以及上月 `月度总结.md` 做原文链接与标题去重；相同原文链接视为硬重复，高相似标题也必须替换，同一期内也不得重复。
- 所有条目都必须与区块链产业相关。通用 TLS、PQC、LLM、CV、数据库、模型后门等非区块链研究，即便技术上很强，也不能混入日报。
- 控制到一个正常上班前十分钟内能读完，但重大新闻可以适度展开。
- `重复项检查结果` 是内部校验输出，不写进正文。

## 去重与历史比对

- 每日候选新闻在成稿前，必须与过去 `7` 天日报以及上月 `月度总结.md` 做历史比对。
- 如果两条内容在事件主体、发生时间、关键事实和影响上高度一致，或语义相似度超过 `90%`，应标记为 `疑似重复`。
- 对于同一事件的跨源报道，只保留一个条目，优先级依次为：
  1. 官方公告、事故复盘、治理提案、论文原文
  2. 安全/取证/研究机构
  3. 主流科技或加密媒体
  4. 中文社区媒体
  5. X 帖子、newsletter、聚合器
- 若某条新闻属于同一事件的新进展，必须在正文里明确写出“这是对前次事件的新增进展”，不能当作全新新闻重复刊发。

## 学术来源范围

- `技术研究` 的原始发表时间必须落在 `T-15` 到 `T`。
- 论文检索必须覆盖官方 CCF 国际学术会议/期刊目录中的相关 A/B 类 venue，至少包含这些类别页：
  - `https://www.ccf.org.cn/Academic_Evaluation/ARCH_DCP_SS/`
  - `https://www.ccf.org.cn/Academic_Evaluation/CN/`
  - `https://www.ccf.org.cn/Academic_Evaluation/NIS/`
  - `https://www.ccf.org.cn/Academic_Evaluation/Cross_Compre_Emerging/`
- 预印本检索必须覆盖：
  - `https://arxiv.org/`，重点关注 `cs.CR`、`cs.NI`、`cs.DC`
  - `https://eprint.iacr.org/`
- 可补充 `OpenReview` 与研究导向的 `SSRN`，也可用高质量第三方技术分析作为线索或条目来源，但最终 `原文链接` 必须回溯到原始出处。
- 每条 `技术研究` 条目都必须包含原文链接、原始发表、摘要速写和 `3` 条核心贡献。
- 如果标题本身较泛，必须在原文页面标题或摘要中看到明确的区块链指向词，例如 `blockchain`、`Ethereum`、`Bitcoin`、`smart contract`、`rollup`、`zkEVM`、`MEV`、`stablecoin`、`wallet`、`validator`、`cross-chain`、`virtual asset` 等。
- `核心贡献` 的固定格式如下，禁止改写成同一行里的长句或顿号串联：

```markdown
- 原文链接：https://example.com
- 原始发表：YYYY-MM-DD
- 摘要速写：一句话摘要
- 核心贡献：
  - 贡献 1
  - 贡献 2
  - 贡献 3
```

## 总结类文件

- `WeekN/本周总结.md`：必须比日刊更详细，连接同周事件，强调时间线、主线演化和连续性。
- `YYYY年M月刊/月度总结.md`：必须加入月内时间线，系统回顾技术、安全、市场、研究四条主线。
- `YYYY年年度总结.md`：年度复盘应是最详细的一层，可加入榜单、时间线和主题盘点。
