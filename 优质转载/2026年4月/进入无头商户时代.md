## 中文学习稿

上周，一个拥有超过 60 项服务的市场开业了，这些服务的设计初衷不是为了让人类使用，而是为了让 AI Agent（人工智能代理）消耗。

其中的服务包括：
- 对有史以来所有的 SEC 申报文件进行全文检索，按查询收费。
- 为遇到专为机器人设计的验证墙的 Agent 提供 CAPTCHA（验证码）破解。
- 根据文档和地址打印并邮寄实体信件。
- 来自 fal.ai 的图像生成，涵盖 600 多个 AI 模型，每次请求的费用仅为几分之一美分。

驱动该市场的协议是由 Stripe 和 Tempo 开发的机器支付协议（Machine Payments Protocol, MPP），它允许 Agent 在单次 HTTP 请求中使用银行卡、稳定币或 Lightning（闪电网络）进行支付。在运行的第一周，894 个 Agent 在该目录中执行了超过 31,000 次交易，价格从每次请求 0.003 美元到 35 美元不等。

这些服务都没有结账页面。它们的目录是机器可读的 schema。价格嵌入在它们的 HTTP 响应中。Agent 读取 schema，发送请求，支付，并在一次交换中接收输出。

商户过去是一个店面。即使商业转向线上，模式依然保持不变：产品图片、结账页面、确认邮件。在电子商务中，“无头（headless）”意味着前端与后端的解耦。而在新的 Agent 经济中，无头意味着完全消除前端。

这就是“无头商户”：一个没有店面、没有账户、没有销售团队的业务。只有一个服务器、一组端点和每个调用的价格。

使之成为可能的支付轨道现在已经上线。x402 和 MPP 各自采取了不同的方法，但都将支付直接嵌入到 HTTP 请求中。Visa 的 CLI 工具将卡片轨道延伸到了终端。这些是驱动无头商户的基础原语。

### 为什么无头商户与众不同

建立传统的软件业务需要网站、结账流程、用户账户、客户服务、订阅管理、计费系统以及销售团队或营销预算来获取客户。而无头商户只需要一个良好的 API 和一层薄薄的中间件。这就是业务的全部。

这很重要，因为买家发生了变化。Agent 带着任务、预算和约束条件到来。它评估端点的文档、定价和可靠性。如果服务符合标准，它就会支付并继续下一步。

**支付即认证。**

Simon Taylor 将其称为“意图经济（intention economy）”：Agent 带着已经形成的意图到来，商户唯一的工作就是履行它。

这反转了构建业务的思维方式。Agent 买家永远不会看到你的网站。它会看到你的 API 文档、定价和运行时间。一个拥有清晰文档和可预测定价的无头商户，几乎总是能击败一个拥有精美网站但 API 表现平庸的商户。

商业过去发生在特定的地方：商店、网站、应用程序。无头商户将商业转移到了“时刻”。在 Agent 需要它不具备的能力的那一刻，交易就发生了。

### 模式的转变

订阅制摊销了计费成本。注册、输入卡片、选择层级、管理续订：所有这些开销的存在是因为对单次 API 调用向人类收取 0.3 美分是不切实际的。但 Agent 可以处理。一个 Agent 每天可以跨越数十个服务，执行成千上万次支付，每次仅需支付几分之一个美分，且无需创建任何账户。

这改变了哪些业务是可行的。一个每次图像生成收费 0.003 美元或每次网页抓取收费 0.01 美元的服务不需要销售团队。它不需要免费层级，也不必担心流失率，因为没有订阅可以取消，也没有需要维护的关系。它只需要足够出色，当 Agent 评估其文档和定价时会被选中。

如果你今天在 API 密钥和订阅之后销售服务，那么该产品一定存在一个按请求收费、不需要账户、并且可以被任何持有钱包的 Agent 发现的版本。那个版本可能会触达你的订阅产品无法触达的客户，因为那些客户（Agent）根本不会注册。Agent 只会直接跳转到下一个端点。

即付即用（Pay-as-you-go）可能会取代一类日益增长的服务订阅模式。不是因为订阅不好，而是因为买家不再需要它们。

### 商户才是核心故事

本月早些时候我曾认为，下一波商业浪潮将由选择稳定币的商户构建，因为传统处理器无法为他们提供承兑支持。从那时起，基础设施的发展速度超出了预期。卡片网络正在将其轨道延伸到 Agent。支持卡片、稳定币和按会话计费等结算模型的新协议已经出现。支付轨道不再是瓶颈。

现在重要的是商户。一个拥有清晰 API、可靠输出和按请求定价的无头商户是一种新型业务，它的成本结构在五年前不可能存在，而其买家群体在一年前还不存在。

Agent 商业中最大的机会不是构建下一个支付轨道，而是构建这些轨道旨在服务的无头商户。下一代商户不会有店面，他们将拥有端点。
